# Report Writing

## 报告结构

正式报告通常包含：

1. 执行摘要：核心判断、置信度、限制和建议动作。
2. 研究问题与边界：范围、时间、地域、回答和不回答什么。
3. 证据地图与方法：主要通道、口径和结构选择理由。
4. 4-7 个主体章节：每章回答一个决策问题。
5. 风险、盲区和验证路径。
6. 结论与建议。
7. 来源索引。

章节来自证据集群，不来自固定目录。空章节不出现。

## 写作规则

- 使用连续分析正文，表格只用于比较或数据分布。
- 表格前说明来源、口径、时间和用途；表格后解释含义。
- 每个主体章节以阶段性结论收束。
- 重大判断解释“为什么”，并使用 `[S1]` 引用。
- 推断明确写“基于公开资料推断”并标注置信度。
- 来源冲突时展示冲突和选择理由。
- 建议必须回扣证据、限制和执行条件。

## 来源索引

Markdown 来源索引使用固定列：

```markdown
| 编号 | 来源 | 发布机构 | 发布日期 | 类型 | 链接 |
|---|---|---|---|---|---|
| S1 | 文件标题 | 机构 | 2026-07-01 | primary | [原文](https://example.com) |
```

编号必须与 evidence YAML 一致。正文统一使用 `[S1]`，不要混用 `[1]`。

## HTML

只使用 `scripts/render_report.py` 和 `templates/report.html.j2`。默认浅色主题，深色主题仅作为同一模板的变量，不维护第二份 HTML。

必须具备：

- 左侧目录和右侧正文。
- 移动端目录折叠到正文上方。
- 表格横向滚动，不撑破页面。
- 中文长词和 URL 可换行。
- 来源引用跳转和返回可用。
- 外链新窗口打开并带 `noopener noreferrer`。
- 不依赖在线字体或外部 CSS/JS。

## 对话交付

只给出：

- 一段核心判断。
- 2-4 个关键限制或验证动作。
- Markdown、HTML 和 evidence YAML 的可点击路径。

不要把整份报告粘贴回对话。
