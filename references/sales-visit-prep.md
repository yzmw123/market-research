# Sales Visit Preparation — Detailed Guide

在 SKILL.md 中定义了销售拜访准备工作流的基本框架。本文件提供更详细的搜索策略、话题设计模式和话术模板。

## 1. 搜索策略详解

### 政策搜索

政策是最容易引发客户兴趣的话题，因为它直接影响客户的预算和项目节奏。

```bash
# 国务院/部委层面
autocli google search "site:gov.cn {行业关键词} 政策 2026" --limit 5
autocli google search "发改委 {行业} 指导意见 通知" --limit 5
autocli google search "工信部 {行业} 行动计划 白名单" --limit 5

# 省级层面
autocli google search "XX省 {行业} 规划 方案 2026" --limit 5
autocli google search "XX省能源局 通知 意见" --limit 5

# 行业监管
autocli google search "{行业} 监管 新规 标准" --limit 5
```

**政治敏感红线**（适用于面向中国的销售场景）：
- 不谈中美竞争、技术脱钩、地缘政治
- 对"安全审查"、"国产替代"等话题点到为止，不要深入
- 用"主管部门"替代"政府"保持中立客观

### 招投标搜索

```bash
# 全国层面
autocli google search "{行业} 招投标 公告 site:gov.cn" --limit 10

# 本地层面（更精准）
autocli google search "XX市 {行业} 采购 招标" --limit 5
autocli google search "{行业关键词} 中标 金额 2026" --limit 10
```

从招投标数据中可以提取的谈资：
- 本地区最近的采购品类和金额（"最近你们这边XX项目挺多的"）
- 中标企业是谁（了解竞争格局或潜在合作伙伴）
- 预算变化趋势（"今年XX方面的预算好像比去年多了"）

### 案例搜索

```bash
# 全国优秀案例
autocli google search "{行业} 数字化 转型 案例 标杆" --limit 10
autocli google search "全国 {行业} 示范项目 试点" --limit 5

# 同类型客户案例（如果有客户画像）
autocli google search "{客户类型} {行业} 项目 落地 经验" --limit 5
```

### 客户自身搜索（如果能定位到具体单位）

```bash
autocli google search "{客户单位名称} 新闻" --limit 5
autocli google search "{客户单位名称} 领导 调研 讲话" --limit 5
autocli google search "{客户单位名称} 合作 签约" --limit 5
```

## 2. 话题设计模式

### 模式 A：请教式切入（最推荐）

这种模式的核心是：**把对方放在"专家"的位置上，你来请教。**

结构：
1. 引用一个你刚查到的行业动态或政策 → 体现你做了功课
2. 问对方"你们这边怎么看/怎么做" → 把话筒交给对方
3. 对方说完后 → 用"这个思路很好"承接 → 自然过渡到你的方案

示例：
```
你：最近看到能源局出了新型储能的指导意见，要求2027年前建成一批示范项目。
   你们这边感受到这个政策的影响了吗？

对方：是有点影响，我们最近也在研究...

你：这个思路很好。我们最近刚好在做储能电站的AI调度方案，跟你们说的方向挺契合的...
```

### 模式 B：案例引路

结构：
1. 引用一个其他地区/企业的案例 → 不直接说"你们也应该做"
2. 问"你们这边有没有类似的情况" → 让对方自己联想到需求
3. 对方如果感兴趣 → 展开讲你的方案如何服务类似场景

示例：
```
你：浙江那边去年搞了个工业互联网平台，专门做设备预测性维护，
   据说一年帮企业省了300多万的停机损失。你们这边设备运维这块现在是什么模式？

对方：我们主要还是人工巡检...

你：那其实跟我们服务的XX客户情况挺像的，他们一开始也是...
```

### 模式 C：新闻带出痛点

结构：
1. 引用行业新闻 → 最好是负面或挑战性的
2. "你们遇到这个问题了吗" → 引发共鸣
3. "我们最近在研究这个方向" → 切入方案

示例：
```
你：上个月XX厂出了个安全事故，调查报告说是巡检不到位导致的。
   你们这边安质环这块压力大吗？

对方：压力当然大，我们现在每天...

你：理解，XX行业确实这个挑战比较普遍。我们最近在做一套智能巡检系统...
```

### 话题开场模板库

| 场景 | 话题引子 |
|---|---|
| 政策变化 | "最近XX政策出来了，行业里讨论挺多的。你实际接触下来，觉得对你们业务影响大吗？" |
| 新技术 | "最近XX技术挺火的，但行业里真正落地的好像不多。你实际接触过的靠谱的有吗？" |
| 项目动态 | "我注意到你们最近在搞XX项目，方便分享一下大概的思路吗？我们在别的项目上也遇到了类似挑战" |
| 行业痛点 | "你们行业一直有XX的问题，我看到不少企业在尝试XX方案。你们这边有没有考虑过？" |
| 区域对比 | "南方那边XX做得挺早的，我上周刚去看了他们的XX系统。你们这边有没有类似的需求？" |
| 预算话题 | "今年XX方面的预算好像整体在涨，你感觉你们这边的投入节奏怎么样？" |

## 3. 常见行业话术示例

### 电力/能源行业

**政策切入**：
- "最近新型电力系统建设提得挺多的，你们这边在分布式能源和微电网这块有什么规划吗？"
- "发改委刚出了虚拟电厂的试点通知，你们有没有参与或者关注？"

**痛点切入**：
- "新能源并网之后调度压力很大吧？我接触的几家电网公司都在头疼这个"
- "双碳考核越来越严了，你们碳排放管理这块现在怎么做的？"

**案例切入**：
- "江苏那边搞了个源网荷储一体化项目，据说调峰效果很好。你们有没有类似的需求？"
- "内蒙的风电基地最近在推智能运维，用无人机+AI巡检。你们这边风机运维现在是什么模式？"

**绝对避免**：
- 不谈煤电退出、就业安置等敏感话题
- 不评价具体能源政策的好坏
- 不对央企的管理方式发表看法

### 制造业

**政策切入**：
- "工信部最近在推智能制造示范工厂，你们有没有在申报或者计划？"
- "专精特新政策今年好像力度更大了，你们符合条件吗？"

**痛点切入**：
- "招工越来越难了，产线上很多岗位年轻人不愿意干。你们这边用工压力大吗？"
- "客户对质量追溯的要求越来越高了，你们质量数据这块现在怎么管理的？"

**案例切入**：
- "富士康在郑州的工厂上了AI质检，据说缺陷漏检率从3%降到了0.1%。你们这边质检现在是怎么做的？"
- "宁波那边有家汽配厂搞了柔性产线，换线时间从4小时降到30分钟。你们多品种切换这块有什么经验吗？"

### 政府/政务

**政策切入**：
- "最近数字政府建设的考核指标好像又更新了，你们这边感受到压力了吗？"
- "信创替换的节奏你们预计会加快吗？我们最近在帮几个单位做适配方案"

**痛点切入**：
- "数据打通这块其实是最大的挑战，各个业务系统之间的壁垒。你们这边进展如何？"
- "财政收紧之后，信息化项目的审批是不是变难了？"

**案例切入**：
- "浙江最多跑一次之后，广东也在推数字政府2.0。你们这边有没有类似的对标项目？"
- "江苏那边的政务大模型应用据说效果很好，一个事项办理时间从3天缩到2小时"

**绝对避免**：
- 不谈政府效率、官僚主义等负面话题
- 不评价具体领导的决策
- 对政治敏感话题（如数据安全、内容审查）点到为止

## 4. 话术质量自检

生成话术后，用以下标准检查：

- [ ] 每个话题都有一个具体的"引子"（不是泛泛的"最近怎么样"）
- [ ] 引子基于真实的公开信息（新闻/政策/招投标/案例）
- [ ] 话题以"问"为主，不是以"讲"为主
- [ ] 请教式话题占总话题的 60% 以上
- [ ] 每个话题都预设了"对方说了之后你怎么接"
- [ ] 没有推销话术（"我们的产品..."不出现在前 5 轮对话）
- [ ] 没有编造客户内部信息
- [ ] 政治敏感话题已过滤
